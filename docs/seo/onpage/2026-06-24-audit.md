Site: Next.js 16 App Router. SEO via Next.js Metadata API (root template in `src/app/layout.tsx`, per-route `metadata` exports). Visible copy in `src/lib/site.ts` and `src/components/**`.

## Cross-route / site-wide

- **OG image:** `metadataBase` is set (`layout.tsx:21`) and `twitter.card` is `summary_large_image` (`layout.tsx:47`), **but no `openGraph.images` / `twitter.images` is defined anywhere.** Social shares (and many LLM link previews) will render with no image despite the `summary_large_image` card. **Add a static OG image** (e.g. `src/app/opengraph-image.png` or `images` in `layout.tsx` openGraph) — Next.js will wire it automatically.
- **Sitemap vs. noindex:** `sitemap.ts` lists `/`, `/skill`, `/about`. `/about` is currently indexable placeholder content (see above). Decide index-or-not, then keep sitemap consistent.
- **`metadataBase` host mismatch:** canonical/OG use `https://www.webmcp.md` (`layout.tsx:18`, `sitemap.ts:3`, `download` n/a) while the brand/domain is `webmcp.md` (`site.ts:6`). Ensure the `www` vs apex choice is the one that 301-resolves, so canonicals don't point at a redirecting host. Verify; low effort.
- **`alternates.canonical`** is set site-wide to the root URL in `layout.tsx:52`. Per-route pages (`/skill`, `/about`) **inherit the homepage canonical** unless they override it — this can cause `/skill` and `/about` to declare `https://www.webmcp.md/` as their canonical, deindexing them. **Verify Next.js behavior:** `alternates.canonical` in the root layout is inherited as-is (not auto-suffixed with the path). If confirmed, add a per-route `alternates.canonical` to `/skill` and `/about`. **This is potentially high-impact** (could be suppressing `/skill` from the index).
